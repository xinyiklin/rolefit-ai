# RoleFit Local Companion 0.1.0

<!-- Curated release-note source. Publication occurs only after every native signing, notarization, installer, checksum, and atomic-release gate succeeds. -->

This is the first signed desktop companion release for RoleFit AI.

## Included

- Native macOS downloads for Apple silicon and Intel.
- A native Windows x64 installer.
- Local setup for Codex, Claude Code, and Antigravity CLI providers.
- Operating-system-encrypted storage for configured OpenAI and Anthropic API
  keys; saved keys are never returned to the browser UI.
- A configurable local RoleFit site port, defaulting to `5181`.
- The existing RoleFit Drafting Desk in the normal browser, served only from
  the companion-owned loopback server.

## Data boundary

RoleFit has no app account, hosted workspace, cloud synchronization, or
telemetry service. Workspace resumes, jobs, tracker records, and generated
artifacts stay in the local RoleFit workspace. Origin-scoped browser storage
may also hold recovery resume/job drafts plus user settings and context; it
never stores API keys. When you run an AI action, the inputs required for that
action are sent directly to the CLI or API provider you selected. CLI
credentials remain owned by that provider; API credentials stored by the
companion remain encrypted on this device.

## Browser extension

The unpacked browser extension requests local access on first use. Approve its
exact browser/profile origin once in the companion; unapproved extensions
cannot analyze or import a posting, and removing the pairing revokes access.

Every downloadable asset is produced by the native platform release job. The
release includes `SHA256SUMS.txt` for independent checksum verification.
